# P007 麒麟项目 — 多商户扫码点餐云打印 SaaS 平台

## 产品简介

麒麟项目是一个**多商户扫码点餐云打印 SaaS 平台**，支持多租户架构。一个平台可以同时服务多个餐饮商户，每个商户可管理多家门店。顾客扫码点餐、后厨自动打印、店长管理店铺，全流程在线化。

## 二、角色详细说明

### 🏛️ 超级管理员 (SUPER_ADMIN)

**平台运营者**，拥有系统最高权限。

- **入口地址**：`http://localhost:5177/auth/admin/login`
- **测试账号**：`admin@qilin.test` / `Admin@2026`
- **登录后跳转**：超管后台 `/admin`

#### 功能列表

| 功能 | 路径 | 说明 |
|------|------|------|
| 租户管理 | `/admin/tenants` | 创建/编辑/删除商户，管理商户状态 |
| 店铺管理 | `/admin/stores` | 全局查看所有门店，切换状态 |
| 用户管理 | `/admin/users` | 创建用户账号，授权为店长 |
| 菜品模板库 | `/admin/menu-templates` | 管理全局菜品模板（34条Pexels图） |
| 打印机管理 | `/admin/printers` | 全局查看所有打印机 |
| 餐桌管理 | `/admin/tables` | 全局餐桌管理 |
| 夜狼打印配置 | `/admin/nightwolf` | 配置云打印规则引擎 |
| 系统设置 | `/admin/settings` | 平台级配置 |

#### 超管可用角色

超管登录后可以通过 `/t/{商户slug}/s/{店铺slug}/menu` 路径直接访问任何商户的任何店铺的管理页面（路由常量化，非硬编码）。

**餐饮品牌所有者**，管理自己名下的所有门店。

- **入口地址**：`http://localhost:5177/auth/tenant/login`
- **测试账号**：`admin1@qilin.test` / `Admin123`（对应商户 php1682）
- **登录后跳转**：商户工作台 `/t/{slug}/dashboard`

#### 功能列表

| 功能 | 路径 | 说明 |
|------|------|------|
| 商户工作台 | `/t/{slug}/dashboard` | 数据概览 |
| 门店列表 | `/t/{slug}/stores` | 查看/创建/编辑门店 |

- 商户只能查看和管理自己名下的店铺

**单店管理者**，管理一家门店的日常运营。

- **入口地址**：`http://localhost:5177/store-admin/login`
- **测试账号**：`storeadmin@test.com` / `Test123456`

#### 功能列表

| 功能 | 路径 | 说明 |
|------|------|------|
| 店长工作台 | `/store-admin` | 店铺卡片列表，选择要管理的店 |
| 菜单管理 | `/store-admin/stores/:storeId/menu` | CRUD菜品、上下架、图片 |
| 订单管理 | `/store-admin/stores/:storeId/orders` | 查看订单、更新状态 |

1. 超管在 `/admin/users` 创建用户账号
2. 超管通过数据库为账号添加 `UserStore` 授权记录（角色 STORE_ADMIN）
3. 店长用创建的账号密码登录

### 🧑‍🤝‍🧑 顾客 (CUSTOMER)

**用餐者**，无需登录。

- **入口地址**：直接扫餐桌二维码或访问链接
- **测试链接**：
  - 堂食点餐：`http://localhost:5177/t/sdsd/s/panhaiping/scan/A01`
  - 外卖打包：`http://localhost:5177/t/sdsd/s/panhaiping/takeaway`

#### 功能列表

| 功能 | 路径 | 说明 |
|------|------|------|
| 堂食点餐 | `/t/{slug}/s/{storeSlug}/scan/{tableCode}` | 扫码进入，选菜下单 |
| 外卖打包 | `/t/{slug}/s/{storeSlug}/takeaway` | 直接进入菜单 |
| 查看订单状态 | 下单后的订单页 | 实时查看制作进度 |
| 历史菜品建议 | 首页 | 根据历史订单推荐 |

#### 顾客流程
1. 顾客进入餐厅 → 扫描桌上二维码
2. 进入菜单页面 → 浏览分类和菜品
3. 选择菜品加入购物车
4. 确认下单 → 订单发送到后厨打印机
5. 在页面查看订单状态（制作中/已完成）
6. 服务员上菜/顾客取餐

## 三、业务操作流程

### 🏛️ 超管 — 新增商户（完整流程）

```
1. 登录超管后台  →  http://localhost:5177/auth/admin/login
2. 创建商户      →  菜单"租户管理" → "创建租户"
   - 填写商户名称、子域名（slug）、联系人信息
3. 创建商户门店  →  进入商户详情 → "创建门店"
   - 填写门店名称、地址、电话等信息
4. 创建店长账号  →  菜单"用户管理" → "创建用户"
   - 填写邮箱、密码、姓名
   - 在数据库中为该用户添加 UserStore 授权
5. 通知店长     →  店长用创建的账号登录店长端
```

### 🏪 商户 — 门店上线（完整流程）

```
1. 登录商户工作台  →  http://localhost:5177/auth/tenant/login
2. 管理菜单       →  选择门店 → 菜单管理
   - 添加分类（川菜、湘菜、主食等）
   - 添加菜品（名称、价格、图片、描述）
   - 设置上下架状态
3. 配置打印机     →  选择门店 → 打印机管理
   - 添加云打印机（序列号、密钥）
   - 测试打印效果
4. 管理餐桌       →  选择门店 → 餐桌管理
   - 添加/编辑餐桌编号
   - 生成餐桌二维码 → 打印张贴
5. 检查设置       →  门店信息、营业时间、联系方式
6. 上线营业       →  顾客扫码即可点餐
```

### 🔧 店长 — 日常运营（完整流程）

```
1. 登录店长端     →  http://localhost:5177/store-admin/login
2. 选择店铺       →  点击要管理的店铺卡片
3. 处理订单       →  订单管理
   - 查看新订单 → 确认接单
   - 后厨制作 → 更新为"制作中"
   - 出餐完成 → 更新为"已完成"
4. 管理菜单       →  菜单管理
   - 下架已售罄菜品
   - 调整菜品排序
   - 更新价格/描述
5. 管理打印机     →  测试打印、更换打印机
6. 查看店铺信息   →  设置页面
```

### 🧑‍🤝‍🧑 顾客 — 到店用餐（完整流程）

```
1. 入座 → 扫描桌上二维码
2. 浏览菜单 → 切换分类、查看菜品详情和图片
3. 加入购物车 → 选择数量
4. 确认下单 → 订单自动打印到后厨
5. 等待上菜 → 页面可查看订单状态
6. 如需加菜 → 再次扫码或叫服务员
7. 用餐完毕 → 前台结账
```

## 六、常见问题

### Q: 如何新增一家门店？
答：超管登录 → 租户管理 → 选择商户 → 创建门店。或商户登录后直接在门店列表创建。

### Q: 如何为店长生成账号？
答：超管登录 → 用户管理 → 创建用户（邮箱+密码）→ 随后需要在数据库中为该用户添加 `UserStore` 记录（角色 `STORE_ADMIN`），才能登录店长端。

### Q: 如何添加云打印机？
答：超管/商户/店长登录后 → 进入打印机管理 → 添加打印机（输入序列号和密钥）→ 测试打印。目前支持飞鹅云打印和夜狼云打印。

### Q: 为什么订单没有自动打印？
答：需在夜狼配置页面 `/admin/nightwolf` 设置打印规则引擎。确保打印机已绑定且状态正常。

### Q: 菜品图片怎么上传？
答：编辑菜品时填写图片URL（支持外链，推荐使用 Pexels CDN 图片），或通过店铺设置的图片上传功能上传 Logo/Header。

### Q: 如何生成餐桌二维码？
答：进入餐桌管理页面 → 点击编辑餐桌 → 点击"生成二维码"，浏览器会弹出提示框显示完整点餐链接。

### Q: 商户如何修改密码？
答：目前密码修改功能尚未实现。需要超管在用户管理中重置密码（功能建设中）。
