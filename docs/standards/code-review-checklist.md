# 代码审查检查清单

## 概述
此检查清单用于确保所有代码变更符合项目规范和最佳实践。

## 通用检查项

### 1. 代码质量 ✅
- [ ] 代码格式符合项目规范 (ESLint/Prettier)
- [ ] 无未使用的导入、变量或函数
- [ ] 函数和方法长度适中（建议 < 50行）
- [ ] 代码有适当的注释（复杂逻辑、算法、业务规则）
- [ ] 无魔法数字/字符串，使用常量或配置

### 2. 类型安全 ✅
- [ ] TypeScript编译无错误 (`npm run check:config`)
- [ ] 类型定义完整且准确
- [ ] 无 `any` 类型（除非必要且有说明）
- [ ] 接口和类型定义在适当的位置
- [ ] 类型兼容性已验证

### 3. 测试覆盖 ✅
- [ ] 新增功能有对应的测试用例
- [ ] 修复的bug有回归测试
- [ ] 测试覆盖关键业务逻辑
- [ ] 测试用例清晰、可读
- [ ] 测试通过率100%

### 4. 安全考虑 ✅
- [ ] 无硬编码的敏感信息（密码、密钥等）
- [ ] 输入验证和输出编码
- [ ] 无SQL注入风险
- [ ] 无XSS漏洞
- [ ] 权限检查完整

## 前端特定检查项

### 1. React组件 ✅
- [ ] 使用函数组件和Hooks（除非有特殊原因）
- [ ] 组件职责单一
- [ ] Props有明确的类型定义
- [ ] 状态管理合理（本地状态 vs 全局状态）
- [ ] 无不必要的重新渲染

### 2. 性能优化 ✅

- [ ] 代码分割合理
- [ ] 无内存泄漏
- [ ] 网络请求有适当的缓存

### 3. 用户体验 ✅
- [ ] 加载状态处理
- [ ] 错误状态处理
- [ ] 空状态处理
- [ ] 响应式设计
- [ ] 无障碍访问支持

### 4. 样式和布局 ✅

- [ ] 样式无硬编码值
- [ ] 响应式布局正确
- [ ] 颜色和字体一致
- [ ] 动画性能优化

## 审查流程

### 1. 提交前检查
```bash
# 运行自动化检查
npm run pre-commit

# 检查结果
- [ ] ESLint检查通过
- [ ] TypeScript检查通过
- [ ] 单元测试通过
- [ ] 集成测试通过
```

### 2. 人工审查
- [ ] 至少1名审查员批准
- [ ] 所有检查项通过
- [ ] 问题讨论和解决
- [ ] 审查意见记录

### 3. 合并后验证
- [ ] 部署到测试环境
- [ ] 功能验证通过
- [ ] 性能测试通过
- [ ] 监控指标正常
