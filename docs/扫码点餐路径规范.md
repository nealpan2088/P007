# 扫码点餐路径规范

## 📋 规范概述

本规范定义了P007麒麟项目扫码点餐功能的路径格式、API端点和实现要求。

## 🎯 核心规范

### **路径格式**
```
/scan/:storeId/:tableId
```

### **参数说明**
- **storeId**: 店铺ID或slug (如: `test-store`, `phoenix-beijing`)
- **tableId**: 餐桌ID或编号 (如: `A01`, `VIP-01`)

### **前端路由**
```jsx
<Route path="/scan/:storeId/:tableId" element={<ScanOrderPage />} />
<Route path="/scan/:storeId" element={<ScanOrderPage />} />
<Route path="/scan" element={<ScanOrderPage />} />
```

### **API端点**
```
GET    /api/public/stores/:storeId           # 店铺信息
GET    /api/public/stores/:storeId/menu      # 店铺菜单  
GET    /api/public/stores/:storeId/tables/:tableId  # 餐桌信息
POST   /api/public/orders                   # 创建订单
GET    /api/public/orders/:orderId/status   # 订单状态
```

## 🛠️ 实现要求

### 1. **使用路由常量**
所有扫码点餐相关代码必须使用 `src/config/scan-routes.ts` 中的常量：

```typescript
import SCAN_ROUTES from '../config/scan-routes';

// 构建扫码点餐URL
const scanUrl = SCAN_ROUTES.routes.utils.buildScanUrl('test-store', 'A01');

// 构建API URL
const apiUrl = SCAN_ROUTES.api.utils.buildStoreMenuUrl('test-store');
```

### 2. **禁止硬编码路径**
❌ 禁止：
```jsx
<Link to="/scan/test-store/A01">扫码点餐</Link>
```

✅ 正确：
```jsx
<Link to={SCAN_ROUTES.getTestScanUrl()}>扫码点餐</Link>
```

### 3. **参数验证**
所有扫码点餐路径必须通过验证：
```typescript
const isValid = SCAN_ROUTES.validateScanPath('/scan/test-store/A01'); // true
const isValid = SCAN_ROUTES.validateScanPath('/scan/invalid'); // false
```

## 🔧 开发指南

### **创建新的扫码点餐链接**
```typescript
// 方法1：使用工具函数
const url = SCAN_ROUTES.routes.utils.buildScanUrl('phoenix-beijing', 'VIP-01');

// 方法2：使用预定义常量
const testUrl = SCAN_ROUTES.getTestScanUrl();    // /scan/test-store/A01
const demoUrl = SCAN_ROUTES.getDemoScanUrl();    // /scan/demo-shop/B02
```

### **调用API**
```typescript
import SCAN_ROUTES from '../config/scan-routes';

// 获取店铺菜单
const menuUrl = SCAN_ROUTES.api.utils.buildStoreMenuUrl(storeId);

// 获取餐桌信息
const tableUrl = SCAN_ROUTES.api.utils.buildTableInfoUrl(storeId, tableId);

// 获取订单状态
const statusUrl = SCAN_ROUTES.api.utils.buildOrderStatusUrl(orderId);
```

### **验证路径**
```typescript
// 验证路径格式
if (!SCAN_ROUTES.validateScanPath(path)) {
  console.error('无效的扫码点餐路径:', path);
  return;
}

// 解析路径参数
const params = SCAN_ROUTES.routes.utils.parseScanUrl(path);
// { storeId: 'test-store', tableId: 'A01' }
```

## ❓ 常见问题

### **Q: 为什么要使用路由常量？**
**A**: 确保一致性、便于维护、避免硬编码错误、支持全局配置变更。

### **Q: storeId和tableId有什么格式要求？**
**A**: 
- `storeId`: URL友好，字母数字+连字符，全局唯一
- `tableId`: 支持字母数字+连字符，在店铺内唯一

### **Q: 如何添加新的测试店铺？**
**A**: 在 `scan-routes.ts` 的 `SCAN_ROUTES` 常量中添加新的测试配置。

### **Q: 验证脚本失败怎么办？**
**A**: 
1. 检查是否使用了硬编码路径
2. 确认导入了正确的路由常量
3. 运行 `node scripts/validate-scan-routes.js` 查看具体错误
